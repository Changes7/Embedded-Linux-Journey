# app_code — 应用层代码

本目录存放运行在 **STM32MP157** Linux 用户空间的应用程序源码。

## 目录用途

- 用户态 C/C++ 应用程序
- 基于 POSIX API 的系统编程示例（进程、线程、文件 I/O、网络等）
- 与硬件交互的用户态程序（通过 `/dev` 节点、`sysfs`、`libgpiod` 等）

## 开发规范

1. **交叉编译**：使用 `arm-linux-gnueabihf-gcc` 工具链，通过各子目录下的 `Makefile` 构建
2. **目录组织**：每个独立应用建议使用单独子目录，包含源码、`Makefile` 及简要说明
3. **部署方式**：编译产物通过 NFS 共享目录部署至开发板，执行 `make deploy` 完成拷贝
4. **编码风格**：遵循 Linux 内核编码风格中的命名与缩进习惯，函数与变量名使用英文
5. **提交前检查**：执行 `make clean` 清理编译产物，勿将二进制文件提交至版本库

## 快速开始

```bash
make          # 编译生成 mp157_app
make deploy   # 部署至 NFS 共享目录
make clean    # 清理编译产物
```
