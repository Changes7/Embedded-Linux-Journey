# driver_code — 底层驱动源码

本目录存放 **STM32MP157** 平台相关的 Linux 内核驱动与设备树配置源码。

## 目录用途

- 字符设备、平台设备、I2C/SPI 等总线驱动模块
- 设备树（Device Tree）片段与 Overlay 配置
- 驱动调试、性能优化与内核模块加载相关实验代码

## 开发规范

1. **内核版本对齐**：驱动代码须与目标板运行的 Linux 内核版本保持一致，在目录或文件头注明内核版本
2. **模块结构**：每个驱动独立子目录，包含 `.c` 源文件、`Makefile`（或 Kbuild）及 `README` 说明
3. **设备树**：`.dts` / `.dtsi` 文件单独存放，修改后需同步更新对应的内核设备树编译流程
4. **编译方式**：使用内核源码树外的 out-of-tree 模块编译，或通过 Yocto/Buildroot 集成至镜像
5. **调试记录**：驱动加载顺序、模块参数及 `dmesg` 关键日志应在对应子目录文档中说明
6. **提交规范**：不提交 `.ko` 等编译产物；敏感硬件参数通过设备树或模块参数配置，避免硬编码
